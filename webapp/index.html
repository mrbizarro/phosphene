Output format

API tokens

Saved locally in panel_settings.json. Never sent anywhere except as auth headers to civitai.com / huggingface.co. Power users can override with CIVITAI_API_KEY / HF_TOKEN env vars; the saved value here wins when both are set.
CivitAI API key —
Required for installing CivitAI LoRAs. Get one at civitai.com/user/account (Account → API Keys → Add).
Hugging Face token —
Required for gated LoRAs (Lightricks HDR + Control LoRAs). Get one at huggingface.co/settings/tokens — read access is enough.

Spicy mode (adult content)

When OFF, the CivitAI LoRA browser hides NSFW models entirely — the "Show NSFW" toggle disappears and any NSFW result the API returns is filtered out. When ON, you can choose per-search whether to include NSFW results. The server enforces this gate even if the client is tampered with.

          Project notes

          The agent's persistent memory across sessions. It reads the tail every turn, and can append new entries via append_project_notes. Edit freely; one Markdown file at state/agent_project_notes.md.

          Agent engine

          The agent uses any OpenAI-compatible chat endpoint. Default is Phosphene Local — a small mlx-lm.server pointed at the bundled Gemma 3 12B IT (the same weights LTX uses as its text encoder, doubling as a chat model). Stronger options: drop a Qwen 3 Coder 30B MLX model into mlx_models/ and pick it below; or point at a cloud endpoint (Anthropic compat, OpenAI, OpenRouter, your LM Studio box on the LAN).
          stopped mlx-lm.server
          Manage Ollama models from the terminal: ollama pull qwen2.5-coder:32b, ollama pull huihui_ai/llama3.3-abliterated:70b, etc. Phosphene only talks to Ollama's /v1/chat/completions endpoint — the model has to already be installed in Ollama.
