# Phosphene perf-breakthrough plan — 2026-05-05

## Goal

Deliver a measurable end-to-end speedup on real LTX renders **without quality
regression**, validated by A/B testing.

Target: **≥1.2× end-to-end on a 20-sec 1024×576 Balanced+Turbo render**, with
**no visible quality loss** (frame-by-frame SSIM ≥ 0.95 vs baseline, audio RMS
within ±1 dB of baseline, no character drift, no temporal flicker).

## Status of prior diagnostic work (already done, see /tmp/phosphene_bench/)

| Hypothesis | Result | Action |
|---|---|---|
| mlx-mfa attention drop-in | 1.00× — MLX SDPA already saturated | rule out |
| `mx.compile` block forward | 1.01× | rule out |
| RoPE cache across steps | ~0.16 s/render | rule out |
| Sliding-window attention | kernel 1.9-4.3× but 58-280% drift | rule out without retrain |
| MLX conv3d on VAE | 1.7-2.8 GB/s effective vs 410 GB/s ceiling | viable Track B |
| Block-skip caching (DeepCache for DiT) | 99.9% per-block skip savings | **viable Track A** |

Profile breakdown (real LTX shapes):
- Video self-attn = 41% (5s) / **64% (20s)** of block forward
- Video FFN = 36% / 21%
- Audio = <2%

## Risk register

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Block-skip drops quality on character continuity | Medium | High | Quality-validate A.4 at multiple schedules; ship as opt-in only |
| Inlined `_call_with_block_skip` prelude diverges from upstream model.py changes | Low (now), High (over time) | Medium | Add a unit test that compares patched output bit-equality vs original on a tiny model |
| Helper integration breaks panel boot | Low | High | Try-except guard the import; verify panel boots with patch unloaded |
| GPU contention if panel is also running | Medium | Low | Stop panel before A/B; document |
| Pinokio Reset wipes patch_block_skip.py from install dir | High | Low | The patch lives in repo root, not install dir — safe |
| CFG/STG renders need separate cache per pass — currently shared | Medium | Medium | The patch DOES key cache per pass label; the sampler integration is what passes the label — must verify it threads through correctly |

## Verification gates (must pass before each step)

**Gate 1 — Smoke (after A.1):** all 7 patch smoke tests pass + samplers/helper still import.

**Gate 2 — Tiny render survives (after A.3):** patch-on render completes without crash; wall-time speedup ≥1.10× on this small config.

**Gate 3 — Quality acceptable (after A.4):** at least ONE schedule gives speedup ≥1.20× with SSIM ≥0.95 mean.

**Gate 4 — Production E2E (after A.5):** 20-sec render at the chosen config matches expected speedup vs STATE.md baseline.

## Data we need from each E2E render

For every render, save:
- Wall time (panel sidecar gives this)
- Output mp4 path
- Schedule + edge_blocks used (write to a sidecar JSON next to mp4)
- Per-frame SSIM/PSNR vs A.4.0 baseline (compute via ffmpeg ssim filter)
- Audio RMS dB
- Frame-mean color delta

Tool: `quality_compare.py` (TBD) takes baseline mp4 + variant mp4, prints metrics + saves a side-by-side strip.

## Open questions to resolve as we go

1. Does the `3of5` schedule cycle correctly across CFG passes (i.e., cond and uncond stay in sync at the outer-step level)?
2. Is the `alternate` schedule too aggressive (50% of compute steps skip) — does cache get stale?
3. Should we EXCLUDE early/late blocks beyond just "edge"? E.g., maybe blocks 0-3 always run AND blocks 44-47 always run, leaving only blocks 4-43 as skip-eligible?
